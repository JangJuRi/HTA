HTML
	- HyperText Markup Language
		* 웹용 문서를 작성할 때 사용되는 마크업(태그)를 정의해 놓은 언어
		* HyperText : 문서의 링크를 클릭하면 연결된 문서로 즉시 이동할 수 있는 문서
		* Markup : 문서를 구상하고 있는 여러 요소를 정의할 때 사용되는 태그
		   	   (문서의 구성요소 : 제목, 내용, 리스트, 표, 그림, 링크 등 ...)

	- HTML 문서의 구조
		<!doctype html>	<!-- HTML문서의 유형(형식)을 선언하는 선언문이다. -->
		<html>	<!-- 이 문서가 HTML문서임을 나타낸다. -->
			<head>
				<!-- 웹 브라우저가 HTML 문서를 해석할 때 필요한 정보를 정의한다. -->
			</head>
			<body>
				<!-- 웹 브라우저에 실제로 표시되는 컨텐츠를 정의한다. -->
			</body>
		</html>
		* HTML 문서는 <html>태그를 하나만 포함할 수 있다.
		* <html>태그는 <head>태그와 <body>태그를 각각 하나씩 포함할 수 있다.
		* <html>태그는 이 문서가 html문서임을 정의한다.
		* <head>태그는 웹 브라우저가 이 문서를 해석하는데 필요한 정보를 제공한다.
		* <head>태그에 정의된 내용은 화면에 표시되지 않는다.
		* <body>태그는 웹 브라우저에 실제로 표시될 내용을 포함한다.
		* <body>태그는 문서의 제목, 내용, 그림, 동영상, 링크, 표 등을 포함한다.

	- HTML 문서의 구성요소
		* 태그
			- 태그는 컨텐츠를 담는 그릇이다.
			- 태그는 태그마다 그 용도가 미리 정해져있다.
			- 태그의 이름은 소문자로 적는다.
			- 태그는 여는태그, 닫는태그가 있다.
				* 여는태그 : <h1> <p> <a>
				* 닫는태그 : </h1> </p> </a>
				* 잘못된 태그 작성 : <H1>, < h1>, < /h1>, </ h1>
			- 태그를 작성할 때는 여는태그와 닫는태그의 짝이 맞게 작성해야한다.
				* 올바른 예
					<h1>주요 뉴스</h1>
					<p>오늘의 뉴스입니다.</p>

				* 잘못된 예
					<h1>주요 뉴스
					<p>오늘의 뉴스입니다.</h1></p>
			- 컨텐츠는 여는태그와 닫는태그 사이에 적는다.
			- 컨텐츠는 텍스트나 다른 태그가 될 수 있다.
				* 컨텐츠가 text인 경우
					<p>텍스트 컨텐츠입니다.</p>

				* 컨텐츠가 다른 태그인 경우
					<div><h1>컨텐츠입니다.</h1></div>

			- 태그들 사이에는 부모태그, 자식태그의 관계를 가지는 태그가 있다.
				* 자식태그는 부모태그 안에서만 사용할 수 있다.
				* 자식태그를 부모태그 안에 적을 때는 들여쓰기한다.
				* 예)
					<head>의 부모는 <html>이다.
					<body>의 부모는 <html>이다.
					<li>의 부모는 <ul> 혹은 <ol>이다.
					<thead>, <tbody>, <tfoot>의 부모는 <table>이다.

				* 올바른 예
					<ul>
						<li>커피</li>
						<li>쥬스</li>
					</ul>

				* 올바르지 않은 예
					<p>
						<li>커피</li>
						<li>쥬스</li>
					</p>
					<div>
						<body>컨텐츠</body>
					</div>

			- 태그 중에는 컨텐츠를 가지지 않는 태그도 있다.
				* 여는태그 닫는태그의 형태로 적지 않는다.
				* 예
					<img /> <hr /> <input />

			- 태그는 하나 이상의 속성을 가지고있다.
				* 속성은 그 태그와 관련된 부가적인 정보, 추가기능을 정의할 때 사용한다.
				* 모든 태그가 공통으로 가지고 있는 속성도 있고, 특정 태그만 가지고 있는 속성도 있다.
				* 속성은 이름과 값으로 구성되어있다.
					<태그 속성명="속성값">
				* 하나의 태그에 동일한 속성명을 여러 번 적을 수 없다.
				* 태그에 여러 개의 속성을 적을 때 특별한 순서는 없다.
					<img src="logo.png" width="50" height="50"/>
					<img width="50" height="50" src="logo.png"/>
					<img height="50" src="logo.png" width="50"/>
					* 전부 유효한 작성법이다.

				* 속성은 본래의 사용목적에 맞게 사용해야한다.
				* 임의의 속성을 마음대로 추가할 수 없다.
				* 어떤 속성은 속성값도 미리 정해진 값들 중에서만 사용해야한다.
					<input type="속성값">
						유효한 속성값 : text, password, checkbox, radio, file, hidden, ...
				* data-*로 시작하는 속성은 임의로 추가할 수 있다.
					<img src="김수영.png" data-job="가수"
							      data-genre="인디"
							      data-company="매직스트로베리사운드"/>

		* 속성

		* 컨텐츠


Block 엘리먼트, Inline 엘리먼트
	- Block 엘리먼트
		* 항상 새로운 줄에서 시작한다.
		* 항상 사용가능한 최대 너비를 차지한다.
		* 블록 엘리먼트의 왼쪽이나 오른쪽에는 다른 엘리먼트가 위치할 수 없다.
		* 엘리먼트의 너비를 조절할 수 있다.
		* 엘리먼트 내의 컨텐츠(자식엘리먼트 혹은 텍스트)의 위치를 조정할 수 있다.
		* 대표적인 태그
			h1~h6, p, ol, ul, li, dl, dt, dd, table, form
			article, aside, section, nav, video, footer

		* div 태그
			* 블록엘리먼트, 인라인 엘리먼트를 그룹으로 묶을 때 사용하는 엘리먼트
			* 용도
				- 화면의 레이아웃 구성
				- 여러 요소를 그룹화해서 일관된 스타일을 적용하기 위해

	- inline 엘리먼트
		* 새로운 줄에서 시작하지 않을 수 있다.
		* 컨텐츠를 표시하는데 필요한 만큼의 너비만 가진다.
		* 엘리먼트의 너비를 조절할 수 없다.
		* 엘리먼트내의 컨텐츠(자식엘리먼트 혹은 텍스트)의 위치를 조정할 수 없다.
		* 대표적인 태그
			a, img, strong, em, small, del, input, select, textarea, button, label

		* span 태그
			* 텍스트의 일부분에 특별한 스타일을 적용하기 위해 해당 텍스트를 감싸는 엘리먼트

	- 주의사항
		* div 엘리먼트는 모든 엘리먼트를 자식으로 포함할 수 있다.
		* 모든 block 엘리먼트는 inline 엘리먼트를 자식엘리먼트로 포함할 수 있다.
		* 모든 block 엘리먼트는 div 엘리먼트를 자식으로 포함할 수 있다.
		* inline 엘리먼트는 반드시 block 엘리먼트의 자식 엘리먼트가 되도록 작성한다.
		* inline 엘리먼트는 block 엘리먼트를 자식 엘리먼트로 포함시키지 말자.
		* <p> 엘리먼트는 다른 block 엘리먼트나 <div> 엘리먼트를 자식 엘리먼트로 포함시키지 말자.


엘리먼트의 id속성과 class 속성
	* id와 class는 모든 엘리먼트가 가지는 공통 속성이다.
	* id와 class의 주요 사용 용도
		- id
			* 자바스크립트 프로그램으로 특정 엘리먼트를 조작하기 위해서 사용된다.
			* 조작대상이 되는 엘리먼트에 id="식별자"를 지정한다.
			* 조작작업 = 컨텐츠 조회, 컨텐츠 변경, 삭제, 추가	
	
		- class
			* 엘리먼트에 스타일을 적용하기 위해서 사용한다.
			* 미리 퍼블리셔가 다양한 스타일을 .식별자{스타일}의 형태로 정의해놓는다.
			* 해당 스타일을 적용하고 싶은 엘리먼트에 class="식별자"를 지정하면 미리 정의된 스타일이 적용되게 된다.

	* id 속성
		- 형식
			<tag id="값">컨텐츠</tag>

		- 특정 엘리먼트를 식별하기 위한 용도로 사용된다.
		- HTML문서 전체에서 동일한 id속성값을 가지는 엘리먼트가 없어야 한다.
		- id속성의 값에는 특수문자를 사용할 수 없다. (_ - 는 사용 가능)
			<tag id="book-list">	O
			<tag id="book_list">	O
			<tag id="book/list">	X (특수문자를 사용할 수 없다.)	
			<tag id="book1">	O
			<tag id="1num">		X (숫자로 시작할 수 없다.)
			<tag id="num list">	X (공백이 포함될 수 없다.)

	* class 속성
		- 형식
			<tag class="값1 값2 값3 ...">컨텐츠</tag>

		- 엘리먼트에 스타일을 적용하기 위한 용도로 사용된다.
		- HTML문서 전체에서 동일한 속성값을 가지는 엘리먼트가 여러 개 있을 수 있다.
			<tag>
				<tag id="box-1" class="panel panel-heading">컨텐츠</tag>
				<tag id="box-2" class="panel panel-body panel-default">컨텐츠</tag>
				<tag id="box-3" class="panel panel-body">컨텐츠</tag>
				<tag id="box-4" class="panel">컨텐츠</tag>
			</tag>
